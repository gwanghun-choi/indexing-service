# PRD(Product Requirements Document) 생성

작업명: $ARGUMENTS

## 절차

### 1. 문제 정의

작업이 필요한 배경과 해결하려는 문제 파악:

- 해결하려는 문제가 무엇인가?
- 현재 상황의 pain point는?
- 이 작업의 목표는?

### 2. 배경 조사

현재 시스템 분석:

- 관련 파일/모듈 확인
- 현재 구현 방식 분석
- 의존성 파악

### 3. 요구사항 정의

- Functional Requirements (기능 요구사항)
- Non-Functional Requirements (성능, 보안, 유지보수성)
- 제약사항 (Constraints)

### 4. 해결 방안 검토

- 대안 검토 (1개 이상)
- 각 대안의 장단점 분석
- 최적 방안 선택 및 근거

### 5. PRD 문서 생성

**파일 위치**: `docs/working_history/YYMMDD/<작업명>_prd.md`

## 주의사항

1. **간결하게 작성**: 필요한 섹션만 사용, 해당 없는 섹션은 생략 가능
2. **측정 가능하게**: 추상적 표현보다 구체적 기준 사용
3. **트레이드오프 명시**: 선택한 방안의 장단점 솔직하게 기록

---

## PRD 수정이 필요한 경우

작업 중 요구사항 변경이나 새로운 제약 발견 시:
1. 변경 사유를 해당 섹션에 기록
2. 영향받는 계획서도 함께 검토
